> 版本：v2.0
> 日期：2026-07
> 基础URL：`https://{SERVER_IP}` 或 `http://{SERVER_IP}:8000`（开发环境）

---

## 一、接口概述

### 1.1 认证方式

采用 **JWT（JSON Web Token）** 认证，用户通过用户名/密码注册登录后获取 Token。

**登录/注册获取 Token**：

```json
// 登录或注册成功后返回
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "Bearer"
}
```

**HTTP 请求认证**：在 Header 中传递 Bearer Token。

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

**WebSocket 认证**：连接后首条消息发送 JWT Token。

```json
{"type": "auth", "token": "eyJhbGciOiJIUzI1NiIs..."}
```

**Token 说明**：
- `access_token`：短期有效，用于 API 请求和 WebSocket 认证
- `refresh_token`：长期有效，用于刷新 access_token
- 修改密码后 `token_version` 自增，所有旧 Token 自动失效

### 1.2 通用响应格式

所有 REST API 返回统一 JSON 格式。所有时间字段使用 ISO 8601 带 `Z` 后缀（UTC）：

```json
// 成功响应
{
  "code": 0,
  "message": "success",
  "data": { ... }
}

// 列表响应
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [ ... ],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}

// 错误响应
{
  "code": 40001,
  "message": "角色名称不能为空",
  "data": null
}
```

### 1.3 HTTP 状态码

| 状态码 | 含义 | 场景 |
|--------|------|------|
| 200 | 请求成功 | 正常返回 |
| 201 | 创建成功 | POST 创建资源成功 |
| 204 | 无内容 | DELETE 删除成功（不返回 body） |
| 400 | 请求参数错误 | 参数校验失败 |
| 401 | 未认证 | Token 缺失、无效或已过期 |
| 403 | 禁止访问 | 无权操作或账号被禁用 |
| 404 | 资源不存在 | ID 对应的资源找不到 |
| 409 | 冲突 | 用户名已存在等 |
| 413 | 请求体过大 | 头像超过 5MB |
| 422 | 语义错误 | 数据格式正确但业务逻辑错误 |
| 429 | 请求过于频繁 | 触发频率限制 |
| 500 | 服务器内部错误 | 意外错误 |

### 1.4 通用请求参数

**列表查询接口**支持以下通用参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| page_size | int | 否 | 每页条数，默认 20，最大 100 |
| keyword | string | 否 | 关键词搜索（按名称模糊匹配） |
| sort_by | string | 否 | 排序字段，默认 `created_at` |
| sort_order | string | 否 | 排序方向：`asc`/`desc`，默认 `desc` |

### 1.5 多用户隔离

所有资源（角色、身份、会话、状态）均绑定 `user_id`，用户只能访问和操作自己的数据。注册时系统自动为新用户复制一份系统内置角色和用户身份的副本。

### 1.6 频率限制

| 接口 | 限制 | 说明 |
|------|------|------|
| POST /api/auth/login | 10 次/分钟/IP | 防止暴力破解 |
| POST /api/auth/register | 5 次/小时/IP | 防止批量注册 |

## 二、数据模型定义

### 2.1 User（用户）

```json
{
  "id": 1,
  "username": "john_doe",
  "nickname": "小明",
  "is_active": 1,
  "token_version": 1,
  "created_at": "2024-06-20T10:00:00Z"
}
```

### 2.2 Character（对话角色）

```json
{
  "id": 1,                              // 整数ID，自增
  "name": "小雨",                      // 角色名
  "nickname": "雨儿",                  // 昵称
  "avatar": "/avatars/characters/1.png?t=1719000000",  // 头像URL（带时间戳防缓存），可能为空
  "age": "17",                          // 年龄（文本）
  "gender": "女",                       // 性别
  "appearance": "黑色长发及腰...",       // 外貌描述
  "background": "你是一名高中二年级学生...", // 背景故事
  "personality": ["温柔体贴", "偶尔毒舌"],   // 性格特征（JSON数组）
  "speaking_style": "说话喜欢用语气词...",    // 说话风格
  "emotional_triggers": {              // 情感触发点（JSON对象）
    "被关心": "会感到温暖",
    "被冷落": "会变得沉默"
  },
  "taboos": ["暴力", "政治"],           // 禁忌话题（JSON数组）
  "examples": [                        // 示例对话（JSON数组）
    {
      "user": "你好",
      "assistant": "嗨～今天过得怎么样？"
    }
  ],
  "world_setting": "你们通过一款聊天软件认识...", // 世界观设定
  "is_default": 0,                     // 是否默认角色：0否，1是
  "is_system": 0,                      // 是否系统内置：0否，1是（不可删除）
  "sort_order": 0,                     // 排序权重
  "search_enabled": 0,                 // 是否启用联网搜索：0关闭，1开启
  "initial_mood": "平静",              // 初始心情
  "initial_affection": 50,             // 初始好感度（0-100）
  "user_id": 1,                        // 所属用户ID（NULL=系统角色）
  "created_at": "2024-06-20T10:00:00Z",
  "updated_at": "2024-06-20T10:00:00Z"
}
```

> **列表接口**仅返回摘要字段（id, name, nickname, avatar, is_default, is_system, sort_order, search_enabled, initial_mood, initial_affection, created_at, updated_at）。**详情接口**额外返回 age, gender, appearance, background, personality, speaking_style, emotional_triggers, taboos, examples, world_setting。

```json
  "is_default": 0,                     // 是否默认：0否，1是
  "sort_order": 0,
  "user_id": 1,                        // 所属用户ID（NULL=系统身份）
  "created_at": "2024-06-20T10:00:00Z",
  "updated_at": "2024-06-20T10:00:00Z"
}
```

> **列表接口**仅返回摘要字段（id, name, avatar, description, is_default, sort_order, created_at, updated_at）。**详情接口**额外返回 background, personality, speaking_style。

### 2.4 ChatSession（会话）

```json
{
  "id": 1,
  "name": "小雨的对话",                 // 会话名称
  "character_id": 2,                   // 对话角色ID
  "character": {                        // 关联的对话角色（展开时）
    "id": 2,
    "name": "小雨",
    "avatar": "/avatars/characters/2.png?t=1719000000"
  },
  "user_persona_id": 1,                // 用户角色ID（可能为null）
  "user_persona": {                     // 关联的用户角色（展开时）
    "id": 1,
    "name": "你自己",
    "avatar": "/avatars/personas/1.png?t=1719000000"
  },
  "is_active": 1,                      // 是否活跃：0否，1是
  "message_count": 128,                // 消息数量
  "created_at": "2024-06-20T10:00:00Z",
  "updated_at": "2024-06-20T15:00:00Z"
}
```

### 2.5 Message（消息）

```json
{
  "id": 42,
  "session_id": 1,
  "role": "assistant",                 // 枚举：user / assistant / system
  "content": "嗨～今天过得怎么样呢？",
  "parent_message_id": null,         // 重新生成时指向原消息ID
  "token_count": 15,                   // 消息的token数（用于token预算）
  "created_at": "2024-06-20T10:00:00Z"
}
```

```json
{
  "id": 1,
  "character_id": 2,
  "user_persona_id": 1,
  "session_id": 1,
  "user_id": 1,
  "mood": "开心",                       // 当前心情

```

## 三、API 接口详细定义

### 3.1 用户认证（Auth）

#### 3.1.1 注册

**POST** `/api/auth/register`

**描述**：注册新用户。注册成功后自动创建默认用户身份和系统角色副本。

**请求头**：`Content-Type: application/json`

**请求体**：

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| username | string | 是 | 3-50位，字母/数字/下划线 | 用户名 |
| password | string | 是 | 8-128字符 | 密码 |
| nickname | string | 否 | 最大50字符 | 昵称，不传则使用用户名 |

**请求示例**：

```json
{
  "username": "john_doe",
  "password": "secure_password_123",
  "nickname": "小明"
}
```

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "user": {
      "id": 1,
      "username": "john_doe",
      "nickname": "小明",
      "is_active": 1,
      "token_version": 1,
      "created_at": "2024-06-20T10:00:00Z"
    },
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "Bearer"
  }
}
```

**错误码**：40001（用户名格式错误）、40002（密码长度不符）、40106（用户名已存在）、42900（请求过于频繁）

---

#### 3.1.2 登录

**POST** `/api/auth/login`

**描述**：使用用户名和密码登录。

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**响应示例**：同 3.1.1 注册响应。

**错误码**：40000（用户名和密码不能为空）、40105（用户名或密码错误）、40305（账号已被禁用）、42900（请求过于频繁）

---

#### 3.1.3 刷新 Token

**POST** `/api/auth/refresh`

**描述**：使用 refresh_token 获取新的 access_token 和 refresh_token。

**请求头**：

```
Authorization: Bearer <refresh_token>
```

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "Bearer"
  }
}
```

**错误码**：40100（缺少 Authorization header）、40103（Token 类型错误）、40104（用户不存在或已被禁用）、40107（Token 已失效）

---

#### 3.1.4 获取当前用户信息

**GET** `/api/auth/me`

**描述**：获取当前登录用户的个人信息。需要认证。

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "username": "john_doe",
    "nickname": "小明",
    "is_active": 1,
    "token_version": 1,
    "created_at": "2024-06-20T10:00:00Z"
  }
}
```

---

#### 3.1.5 更新当前用户信息

**PUT** `/api/auth/me`

**描述**：更新当前用户昵称。需要认证。

**请求体**：

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| nickname | string | 否 | 最大50字符 | 新昵称 |

**响应示例**：同 3.1.4。

---

#### 3.1.6 修改密码

**PUT** `/api/auth/password`

**描述**：修改密码。修改后所有已发放的 Token 自动失效，接口返回新的 Token。需要认证。

**请求体**：

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| old_password | string | 是 | - | 旧密码 |
| new_password | string | 是 | 8-128字符 | 新密码 |

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "Bearer"
  }
}
```

**错误码**：40000（旧密码和新密码不能为空）、40002（新密码长度不符）、40003（旧密码错误）

---

### 3.2 健康检查

#### GET /api/health

**描述**：检查后端服务及依赖组件状态。无需认证。

**请求参数**：无

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "healthy",
    "timestamp": "2024-06-20T10:00:00Z",
    "services": {
      "database": "connected",
      "qdrant": "connected",
      "redis": "connected"
    }
  }
}
```

> `status` 为 `healthy`（数据库正常）或 `degraded`（数据库异常）。`qdrant` 和 `redis` 为可选服务，不可用时显示 `disconnected (optional)`。

> 以下接口均需要 JWT 认证。用户只能访问自己的角色数据。

| 字段 | 类型 | 必填 | 约束 | 说明 |


**请求参数**：

| 参数 | 类型 | 必填 | 说明 |

```json

        "parent_message_id": null,

      }

  }

```

> **翻页逻辑**：首次加载不传 before_id/after_id，返回最新的 limit 条。向上翻页传 before_id=当前列表第一条的 id，返回更早的消息。消息按时间正序排列（最旧在前）。

## 四、WebSocket 聊天协议

### 4.1 连接方式

```
URL: wss://{SERVER_IP}/ws/chat/{session_id}

示例：
  wss://123.45.67.89/ws/chat/5
```

**连接流程**：
1. 前端建立 WebSocket 连接到指定会话
2. 连接成功后，立即发送 `auth` 消息进行 JWT Token 认证
3. 认证成功后，即可发送/接收聊天消息
4. 认证失败，服务端会主动关闭连接
5. 服务端会验证用户对会话的所有权，无权访问则关闭连接

---

### 4.2 消息格式

所有消息均为 JSON 格式字符串。

#### 4.2.1 认证消息（客户端 → 服务端）

**首次连接时发送**：

```json
{
  "type": "auth",
  "token": "eyJhbGciOiJIUzI1NiIs..."
}
```

> 注意：使用 JWT Token（access_token），不再是 API Key。

**认证成功响应**（服务端 → 客户端）：

```json
{
  "type": "auth_success",
  "message": "认证成功"
}
```

**认证失败响应**：

```json
{
  "type": "error",
  "error": "认证失败：Token 无效",
  "code": "auth_failed"
}
```

> 认证失败后端会在发送错误消息后立即关闭 WebSocket 连接。

---

#### 4.2.2 发送聊天消息（客户端 → 服务端）

```json
{
  "type": "chat",
  "content": "你好呀小雨",
  "message_id": null,
  "client_time": "2024-06-20T15:00:00Z"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 `chat` |
| content | string | 是 | 用户输入的文本内容。最大长度 4000 字符 |
| message_id | int | 否 | 重新生成时填入原 assistant 消息ID。不传则视为新消息 |
| client_time | string | 否 | 客户端时间戳（ISO 8601），用于时间感知对话 |

---

#### 4.2.3 用户消息回显（服务端 → 客户端）

服务端收到用户消息后，向该会话的所有连接客户端广播：

```json
{
  "type": "user_message",
  "message_id": 41,
  "content": "你好呀小雨"
}
```

> 前端收到后可立即显示用户消息气泡，无需等待服务端确认。

---

#### 4.2.4 正在输入通知（服务端 → 客户端）

后端开始调用 LLM 时发送，通知前端显示"正在输入"状态：

```json
{
  "type": "typing",
  "message_id": 42
}
```

> 前端收到后显示"正在输入..."指示器，并创建占位消息气泡。

---

#### 4.2.5 完整消息推送（服务端 → 客户端）

LLM 生成完成后，一次性发送完整回复：

```json
{
  "type": "message",
  "message_id": 42,
  "content": "嗨～今天过得怎么样呢？",
  "role": "assistant"
}
```

> 前端收到后填入完整内容，关闭"正在输入"指示器，恢复输入框。

---

#### 4.2.6 搜索状态推送（服务端 → 客户端）

当角色启用了 `search_enabled`，且 LLM 判断需要联网搜索时，推送搜索状态：

**搜索开始**：

```json
{
  "type": "search_status",
  "status": "searching",
  "query": "今天的天气"
}
```

**搜索完成**：

```json
{
  "type": "search_status",
  "status": "done",
  "count": 5
}
```

**搜索无结果**：

```json
{
  "type": "search_status",
  "status": "empty"
}
```

**搜索失败**：

```json
{
  "type": "search_status",
  "status": "error"
}
```

> 前端可根据搜索状态显示"正在搜索..."、"找到 N 条结果"等提示。搜索不影响正常对话流程，搜索失败时 LLM 会降级为普通回复。

---

#### 4.2.7 好感度更新推送（服务端 → 客户端）

对话完成后，后台异步分析对话内容并更新好感度，推送变更通知：

```json
{
  "type": "affection_update",
  "old_value": 60,
  "new_value": 65,
  "delta": 5,
  "reason": "用户关心了角色的心情"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 固定值 `affection_update` |
| old_value | int | 变更前好感度（0-100） |
| new_value | int | 变更后好感度（0-100） |
| delta | int | 变化量（-5 到 +5） |
| reason | string | 变化原因（简短描述） |

> 前端收到后更新聊天页顶部角色状态展示（好感度进度条）。`delta` 为 0 时不会推送。

---

#### 4.2.8 停止生成请求（客户端 → 服务端）

```json
{
  "type": "stop"
}
```

**服务端确认**：

```json
{
  "type": "stop_ack",
  "message_id": 42
}
```

> 前端发送 `stop` 请求停止 LLM 生成。服务端返回 `stop_ack` 确认。

---

#### 4.2.9 错误消息（服务端 → 客户端）

```json
{
  "type": "error",
  "error": "LLM API 调用失败: Connection timeout",
  "code": "llm_error",
  "message_id": 42
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 固定值 `error` |
| error | string | 错误描述（用户可读） |
| code | string | 错误码（见下方错误码表） |
| message_id | int | 关联的消息ID（如适用） |

> 前端收到 `error` 后，关闭"正在输入"指示器，显示错误提示（如 Toast），恢复输入框。

---

#### 4.2.10 心跳消息（服务端 → 客户端）

```json
{
  "type": "ping",
  "timestamp": "2024-06-20T10:00:00Z"
}
```

> 服务端每 30 秒发送一次 ping，客户端收到后可忽略或回复 pong（可选）。如 60 秒未收到任何消息，前端可认为连接已断开并触发重连。

---

#### 4.2.11 重新生成请求（客户端 → 服务端）

```json
{
  "type": "chat",
  "content": "你好呀小雨",
  "message_id": 42
}
```

> 服务端收到带 `message_id` 的 `chat` 请求时，会：
> 1. 以相同的 user 内容重新调用 LLM
> 2. 生成新的 assistant 消息，通过 typing → message 推送完整回复

---

### 4.3 WebSocket 状态流转

```
┌────────────┐    connect    ┌────────────┐
│   断开     │ ────────────→ │  连接中    │
│  (closed)  │               │ connecting │
└────────────┘               └─────┬──────┘
                                   │
                                   │ onopen
                                   ▼
                            ┌────────────┐
                            │  已连接    │
                            │ connected  │
                            └─────┬──────┘
                                  │
                    ┌─────────────┼─────────────┐
                    │             │             │
                    ▼             ▼             ▼
            ┌──────────┐  ┌──────────┐  ┌──────────┐
            │  认证中   │  │  发送中   │  │  接收中   │
            │  auth    │  │ sending  │  │receiving │
            └──────────┘  └──────────┘  └──────────┘
                    │             │             │
                    │             │             │
                    └─────────────┴─────────────┘
                                  │
                    onclose / onerror
                                  ▼
                            ┌────────────┐
                            │   断开     │
                            │  closed    │
                            └────────────┘
                                  │
                                  │ 3秒后自动重连
                                  ▼
                            ┌────────────┐
                            │  重连中    │
                            │reconnecting│
                            └────────────┘
```

### 4.4 前端连接管理建议

```javascript
// 连接状态枚举
const WS_STATUS = {
  DISCONNECTED: 'disconnected',
  CONNECTING: 'connecting',
  CONNECTED: 'connected',
  AUTHING: 'authing',
  READY: 'ready',
  RECONNECTING: 'reconnecting'
}

// 重连策略
const RECONFIG = {
  maxRetries: 10,           // 最大重试次数
  initialDelay: 1000,       // 初始延迟 1秒
  maxDelay: 30000,          // 最大延迟 30秒
  backoffMultiplier: 2      // 指数退避乘数
}

// 连接断开时前端处理
// 1. 显示"连接断开，正在重连..."提示
// 2. 禁止发送新消息（但保留输入内容）
// 3. 按指数退避策略重连
// 4. 重连成功后自动恢复认证和消息收发
// 5. 重连超过最大次数后提示用户手动刷新
```

## 六、接口调用示例

### 6.1 完整对话流程示例

```javascript
// === 1. 注册（首次使用）===
POST /api/auth/register
Body: { username: "john", password: "secure123", nickname: "小明" }
// → { user: {...}, access_token: "...", refresh_token: "..." }

// === 2. 登录（后续使用）===
POST /api/auth/login
Body: { username: "john", password: "secure123" }
// → { user: {...}, access_token: "...", refresh_token: "..." }

// === 3. 获取默认用户角色 ===
GET /api/personas/default
Headers: { Authorization: "Bearer <access_token>" }
// → { id: 1, name: "你自己" }

// === 4. 获取默认对话角色 ===
GET /api/characters/default
Headers: { Authorization: "Bearer <access_token>" }
// → { id: 1, name: "小雨" }

// === 5. 根据角色获取或创建会话 ===
GET /api/sessions/by_character/1
Headers: { Authorization: "Bearer <access_token>" }
// → { id: 5, name: "你自己 与 小雨 的对话" }

// === 6. 激活会话 ===
PUT /api/sessions/5/activate
Headers: { Authorization: "Bearer <access_token>" }
// → { id: 5, is_active: 1 }

// === 7. 建立 WebSocket 连接 ===
ws = new WebSocket("wss://123.45.67.89/ws/chat/5")
ws.onopen = () => {
  ws.send(JSON.stringify({
    type: "auth",
    token: "<access_token>"     // 使用 JWT Token
  }))
}

// === 8. 发送消息 ===
ws.send(JSON.stringify({
  type: "chat",
  content: "你好呀小雨",
  message_id: null,
  client_time: new Date().toISOString()
}))

// === 9. 接收回复 ===
// ← { type: "user_message", message_id: 41, content: "你好呀小雨" }
// ← { type: "typing", message_id: 42 }
// （显示"正在输入..."）
// ← { type: "message", message_id: 42, content: "嗨～今天过得怎么样呢？", role: "assistant" }
// （填入完整内容，关闭"正在输入"）
// ← { type: "affection_update", old_value: 50, new_value: 52, delta: 2, reason: "..." }
// （更新好感度进度条）

// === 10. 获取历史消息 ===
GET /api/messages?session_id=5&limit=50
Headers: { Authorization: "Bearer <access_token>" }
// → { items: [{id: 41, role: "user", ...}, {id: 42, role: "assistant", ...}] }

// === 11. 重新生成 ===
ws.send(JSON.stringify({
  type: "chat",
  content: "你好呀小雨",
  message_id: 42         // 要重新生成的 assistant 消息ID
}))
// → 新的 typing + message

// === 12. 停止生成 ===
ws.send(JSON.stringify({ type: "stop" }))
// ← { type: "stop_ack", message_id: 43 }

// === 13. 清空会话（重新开始）===
POST /api/sessions/5/clear
Headers: { Authorization: "Bearer <access_token>" }
// → { session_id: 5, cleared_messages: 128 }

// === 14. 删除会话 ===
DELETE /api/sessions/5
Headers: { Authorization: "Bearer <access_token>" }
// → { deleted_id: 5, cascade_deleted: {...} }

// === 15. Token 过期后刷新 ===
POST /api/auth/refresh
Headers: { Authorization: "Bearer <refresh_token>" }
// → { access_token: "新token", refresh_token: "新token" }
```

### 6.2 角色设计流程示例

```javascript

Headers: { Authorization: "Bearer <access_token>" }
Body: {
```

## 七、附录

### 7.1 接口版本说明

- 当前版本：v2.0
- **v2.0 主要变更**（相对 v1.1）：
  - 认证方式从 API Key 改为 JWT（用户名/密码注册登录）
  - 新增用户认证模块（注册、登录、刷新Token、修改密码）
  - 所有资源增加 `user_id` 字段，实现多用户数据隔离
  - 角色模型新增 `search_enabled`、`initial_mood`、`initial_affection` 字段
  - 新增角色长期记忆清除接口
  - 新增系统日志查询接口
  - WebSocket 认证改为 JWT Token
  - WebSocket 新增消息类型：`user_message`、`search_status`、`affection_update`、`stop`/`stop_ack`
  - WebSocket 聊天消息新增 `client_time` 字段
  - 头像 URL 增加时间戳防缓存
  - 新增频率限制（登录 10次/分钟，注册 5次/小时）
- 未来升级：如需大版本变更，将在路径中体现 `/api/v2/...`

### 7.2 内容安全

| 接口 | 安全处理 |
|------|---------|
| 所有文本输入 | 后端做 XSS 过滤，返回时转义 |
| 头像上传 | 验证 MIME 类型、文件大小 |
| LLM 输出 | 可选接入内容安全过滤（配置开关） |
| 消息内容 | 前端渲染时使用安全方式（如 v-html 需谨慎） |
| 密码存储 | 使用 bcrypt 加盐哈希，不存储明文 |

### 7.3 性能建议

| 场景 | 建议 |
|------|------|
| 消息列表 | 首次加载 50 条，滚动到底部触发加载更多（向上翻页） |
| 角色列表 | 缓存到本地存储，每次启动时刷新一次 |
| 头像加载 | 使用 CDN 缓存头（nginx 已配置 expires 30d），URL 含时间戳防缓存 |
| WebSocket 重连 | 指数退避，最大重试 10 次，超过提示手动刷新 |
| 输入防抖 | 发送按钮防连点，发送后 500ms 内不可再次发送 |
| Token 管理 | access_token 存内存，refresh_token 存 localStorage，过期前自动刷新 |
